### Scenario: Create, list, and delete an employee end-to-end
**Area**: Employees API + Employees page
**Type**: functional
**Preconditions**: backend running, DB seeded with `--count 100 --reset`,
frontend running against the backend.
**Steps**:
1. Open `/employees`.
2. Click "New Employee", fill the form with full_name "Jane Doe", job_title
   "Engineer", country "in", salary "50000.00", submit.
3. Wait for the new row to appear in the table.
4. Click Delete on the new row and accept the browser confirm.
**Expected Result**: row appears with country "IN", salary "50000.00",
"Active" = Yes. After delete, the row is gone and the API returns 404
on `/employees/{id}` for that id.
**Verification**: refresh the page; total count in `X-Total-Count` header
matches the visible row count + filters.

### Scenario: 10,000-row seed completes in well under the 5s budget

**Type**: performance
